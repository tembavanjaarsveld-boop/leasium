# Leasium — 60-Second Launch Video Script (v2)

General-audience launch teaser. Hero: Smart Intake. Now covers tenant portal/self-service, maintenance-to-work-order flow, and payments. Voiceover + matching overlay text per scene. Total VO ≈ 135 words ≈ 56 seconds.

**Before recording:** run `make demo-seed` and record against the fictional Harbour Lane / Rivergum portfolio locally (or a demo org) — never the live SKJ register, which contains real tenant data. Clean browser profile, hide bookmarks, 1920×1080, 125% zoom.

---

## Scene 1 — Hook (0:00–0:05)

**Screen:** Quick cuts: lease PDF, spreadsheet, tenant text message, overdue invoice.

**VO:** "Self-managing rentals shouldn't mean spreadsheets, paperwork, and chasing everyone."

**Overlay:** `Spreadsheets. Paperwork. Chasing.`

---

## Scene 2 — Smart Intake hero (0:05–0:20)

**Screen:** **Smart Intake.** Drag a lease in → extraction review panel (values, confidence, source highlights) → approve → **Apply** → records appear.

**VO:** "Drop in a lease — Leasium's AI reads it. Rent, dates, clauses — extracted with sources shown. You review, you approve."

**Overlay (sequenced):**
1. `Drop in any document`
2. `AI extracts it — sources shown`
3. `You review. You approve.`

---

## Scene 3 — Portfolio appears (0:20–0:27)

**Screen:** **Properties** — rows with owner chips and images → click into a property: units, tenants, leases populated.

**VO:** "Your whole portfolio, built from documents — not data entry."

**Overlay:** `Your portfolio, built from documents`

---

## Scene 4 — Tenant portal + maintenance (0:27–0:39)

**Screen:** Switch to the **tenant portal** (phone-frame it in edit). Tenant views lease and balance, then submits a maintenance request with photos. Cut to operator **Work** view: the request lands as a work order, gets assigned, contractor notified.

**VO:** "Tenants get their own portal — leases, balances, and maintenance requests with photos that land straight in your work queue, assigned and tracked to completion."

**Overlay (sequenced):**
1. `Tenants self-serve`
2. `Photo → work order → contractor notified`

---

## Scene 5 — Money flow (0:39–0:50)

**Screen:** **Billing Readiness / Xero** — invoice drafts → posting preview → explicit Approve. Then payment reconciliation preview: payments matched against invoices, Apply.

**VO:** "Invoices flow to Xero, and incoming payments reconcile themselves — previewed first, posted only when you say so."

**Overlay (sequenced):**
1. `Invoices → Xero`
2. `Payments reconcile themselves`
3. `Approved by you`

---

## Scene 6 — Close (0:50–0:60)

**Screen:** Dashboard wide shot (arrears, critical dates, compliance at a glance) → fade to Leasium logo.

**VO:** "Arrears, compliance, deadlines — all watched for you. Leasium. Property management that reads the paperwork."

**Overlay:** `Leasium` → `leasium.ai`

---

## Recording shot list (in order)

1. Clutter b-roll: PDF, spreadsheet, tenant text (4s; can stage after)
2. Smart Intake: upload → review panel → Apply (record slow, speed 1.5–2× in edit)
3. Properties list → property detail (units/tenants/leases)
4. Tenant portal on a narrow window/responsive view: lease + balance → maintenance request with photo upload
5. Operations/Work: the new request as a work order → assignment → contractor notice (show the review-first send, don't fire a real SMS)
6. Billing Readiness: draft → posting preview → Approve → payment reconciliation preview → Apply
7. Dashboard wide shot (hold 3s for logo fade)

## Edit notes

- Scene 4's portal footage looks best in a phone mockup frame against brand background.
- Cut on clicks; overlays sit ≥1.5s.
- Music: build through Scene 2, hit on "Apply," second smaller hit when the work order appears in Scene 4.
- This cut is dense. If it feels rushed in the edit, the fallback is a 75–90s "full tour" version — cut nothing, just breathe. For a strict 60s, trim Scene 3 first (merge its line into Scene 2), never Scene 2 or 4.
