# Leasium UX Review — 2026-05-22

**Reviewer brief:** act as a senior product designer who lives between Linear, Notion, Stripe, Vercel, and Re-Leased. Audit Leasium honestly. The goal is a product where "users feel like they do little to get a lot." SKJ Capital is using Leasium internally for the first 6 months, so external/customer-facing polish (branded templates, tenant-portal aesthetics) is intentionally deprioritised here — every recommendation below is about making the internal operator experience punch above its weight.

This review reads the current codebase, the existing [design source of truth](leasium-codex-design-source-of-truth.md), and the Remba-pending entries in [design-governance.md](design-governance.md). It is not Remba-approved itself — it is an input for the next Remba review.

---

## 1. What the bar looks like in 2026

Three reference points define the bar for operator-facing SaaS right now:

**Linear** (project management): keyboard-first. `Cmd+K` is just the front door. Power users live on `G I` (Inbox), `G B` (Backlog), `O P` (Open project picker), `C` (Create issue), `X` (Multi-select), `Shift+↑/↓` (Range select). Dark-first; light mode feels secondary. UI is dense without feeling crowded — calm typography, opinionated greys, tiny chips. Right-side detail panels replace page jumps. Bulk actions everywhere. ([Linear changelog](https://linear.app/changelog/2026-03-12-ui-refresh))

**Stripe Dashboard** (financial operator): metric cards with trend indicator + sparkline. Tables are the hero. Filter chips persist as URL params so views are shareable. Every list row is one click from a right-side drawer for editing.

**Notion** (knowledge ops): same data set rendered as Table / Board / Calendar / Gallery / Timeline — operators pick the view that fits their task. Inline editing is the norm.

**Vercel**: ruthless restraint. The dashboard tells you exactly what changed since you last looked. Zero noise. ([Sources: 925studios SaaS dashboard analysis](https://www.925studios.co/blog/saas-dashboard-design-examples-2026), [Orbix B2B SaaS dashboard trends](https://www.orbix.studio/blogs/b2b-saas-dashboard-design-examples))

**Re-Leased Credia** (direct competitor): three AI surfaces embedded in normal workflow:
- **Credia Extract** — drag-and-drop document extraction, review-first apply.
- **Credia Advise** — natural-language Q&A over leases ("When does the rent review on Acme happen?") with source citations.
- **Credia Action** — reads inbox, suggests maintenance tasks, reminders, work orders.
Performance claim: lease data entry from weeks to hours, maintenance triage from 5 minutes to 30 seconds. ([Credia AI](https://www.re-leased.com/artificial-intelligence), [Re-Leased AI guide 2026](https://www.re-leased.com/software/best-ai-powered-property-management-platforms-2026-guide))

**AppFolio Performance Platform** (other direct competitor): AI leasing assistant saves operators "10-15 hours per week on routine tasks" via autonomous task execution. ([AppFolio 2026](https://www.appfolio.com/blog/best-property-management-software-for-leasing-2026/))

The common thread: **the work moves toward the operator, not the other way around.** AI surfaces the next action; tables surface the next row; details surface the next field — and the operator approves rather than authors.

## 2. What Leasium is already doing well

Honest credit before the critique:

| Strength | Why it matters |
|---|---|
| **Review-first automation** is baked into the language, the routes, and the audit history. Source-document references, confidence levels, approve/edit/ignore controls — this matches the modern "AI assists, human approves" pattern that Re-Leased's Credia is selling. | Trust. The bar Linear and Notion don't have to meet because their actions aren't financial. Leasium does. |
| **Status chips** are consistent: Ready (teal), Needs review (blue), Blocked (red), Awaiting (amber). The palette in the [design source of truth](leasium-codex-design-source-of-truth.md) is restrained and modern (Inter/Geist, calm slate neutrals, blue + teal). | Visual language is the cheapest way to scan a screen; Leasium uses it. |
| **Disclosure pattern** for receipt evidence / message previews / provider history is mature. The work-notification stack and contractor channel receipts both use it. | Matches the progressive-disclosure pattern that Stripe and Linear use to keep dashboards calm. |
| **Cmd+K command palette** exists with a curated action list ("Search tenant", "Upload lease", "View billing blockers"). | This is the Linear hallmark and Leasium has the bones in place. |
| **Empty states** — 65 separate `<EmptyState>` usages in `apps/web/src`. Indicates discipline around the "every empty state should teach" principle. | This is a thing teams miss; you didn't. |
| **Filter pills** with live counts on the Properties table (just shipped) are the right pattern. | Modern operator-SaaS norm. |
| **Source-trail evidence drawer** for property apply history is the kind of audit pattern that property managers actually rely on at scale. | Tight, in-context, factual. |
| **Brand discipline.** No "revolutionary AI". No stock-photo handshakes. Calm copy. Operator language. | Most B2B SaaS gets this wrong. Leasium doesn't. |

## 3. Where the gap is widest

These are the gaps where Leasium feels meaningfully behind the modern bar. Listed roughly in order of "would-an-operator-notice".

### 3.1. Navigation: top-bar instead of sidebar

Every modern operator-facing SaaS uses a 240-280px left sidebar — Linear, Notion, Vercel, Stripe Dashboard, Re-Leased, AppFolio. Leasium has a wrapping top-bar (`apps/web/src/components/app-shell.tsx`). The top-bar:

- Doesn't scale past ~8 items without wrapping (which is happening already at narrow widths per the `flex-wrap` in the nav).
- Pushes the workspace switcher into a separate row.
- Wastes vertical pixels on every page.
- Doesn't have room for context (active workspace badge, environment indicator, AI-suggestions widget).
- Doesn't feel like a "command centre" — the design source of truth explicitly calls for `Background: #0B1020` sidebar.

**Effort to fix:** medium. The current nav items list and active-state logic move into a sidebar component; routes don't change. Adds room for a Notion-style workspace switcher, a calm "Today" header, and future AI surfaces.

### 3.2. No AI Q&A — biggest competitive gap

Re-Leased Credia Advise lets an operator type *"When does the Acme lease expire?"* and get an answer with a clause citation. Leasium has all the data and even has OpenAI/SerpAPI plumbing, but no Q&A surface. This is the single biggest "do little, get a lot" lever available to SKJ.

Concretely, a chat field in the Cmd+K palette or a small floating "Ask" widget that can:
- Answer expiry/review/option/rent-review questions over the loaded portfolio.
- Cite the source lease + clause when extracted from a document.
- Suggest the next action ("This lease expires in 23 days — open the renewal flow?").

Internal use accelerates this: with no external customers, the answers can be approximate-but-source-cited without legal exposure.

**Effort to fix:** larger (multi-commit). Realistic v1 in 2-3 commits: a backend `/ai/ask` endpoint that does retrieval over the operator's entity (properties, leases, obligations, maintenance) + an LLM call with citations, plus a floating widget on the dashboard.

### 3.3. Tables are read-only; modern equivalents are inline-editable

Stripe, Notion, Linear all let you click into a row, edit a value inline, and tab to the next field. Leasium tables (Properties, Tenants, Operations) require navigating to a separate form or modal for almost every edit. Examples I noticed:
- Property row `Pencil` button → modal for ABN/contact fields.
- Tenant row → goes to `/tenants/[id]` for any change.
- Lease editing → opens a separate panel.

**Effort to fix:** medium per surface. Most useful for the small repeated edits operators do daily — billing email, owner ABN, lease notes, due dates.

### 3.4. No right-side detail panels (except one)

The design source of truth calls for right-side panels for tenants, leases, properties, documents, billing blockers, tasks. The codebase has exactly one: `MaintenanceDetailPanel` in `/operations`. Everything else is a full-page navigation. Each nav costs a query refetch and breaks scroll position.

**Effort to fix:** medium. Build a generic `<DetailDrawer>` (drawer behaviour, sticky header, primary action footer) and migrate Tenants and Properties off `/[id]` routes onto drawers. The detail routes remain for deep-link sharing.

### 3.5. Dashboard is metric cards + lists, not a "today cockpit"

The design source of truth calls for "Today in lease operations" as the first section. The current dashboard opens with metric cards (Properties, Tenants, Operations, Onboarding, Smart Intake, Documents). Useful but flat — a metric is data, not a next action. Compare to Vercel's "X deployments in last 24h, Y waiting on review" or Stripe's "Today: $X collected, Y disputes need response".

What Leasium has now:
- Metric cards (counts only — no deltas, no sparklines)
- DashboardCommandCenter (lists pending intake/billing/operations work)
- Upcoming lease events (just shipped — good)
- Smart Intake panel
- Various supporting sections

What it's missing:
- **Trend deltas** on metric cards (Stripe-style "↑ 12% vs last week").
- **Sparklines** on metric cards (Stripe-style 30-day mini-chart).
- **"What changed since you last looked"** banner — Vercel's signature move.
- **Cross-property activity feed** — "Maintenance closed for Queen Street, Insurance certificate uploaded for Bright Cafe."

**Effort to fix:** small (deltas, sparkline placeholders) to medium (activity feed needs a backend endpoint).

### 3.6. No saved views / persistent filters

Operators repeat the same filter combinations daily — "Vacant properties in Brisbane" or "Tenants with active leases over $100k/yr" or "Open maintenance work orders assigned to me." Notion and Linear let you save these as named views. Leasium has filter pills (just shipped) but no save button and no URL persistence beyond `owner_tag`.

**Effort to fix:** small (URL params for all filters) to medium (save named views per operator).

### 3.7. Keyboard shortcuts stop at Cmd+K

Linear's keyboard model:
- `G I` (go to Inbox), `G B` (Backlog), `G V` (Cycle) — view switching
- `O P` (open project picker), `O F` (favourites)
- `C` (create), `S` (slash menu), `X` (select), `Esc` (back)
- `Shift+↑/↓` (multi-select), `Tab` (next field)

Leasium has Cmd+K + Esc. Operators who would otherwise live on the keyboard are forced back to the mouse. For a daily-use internal tool this is meaningful productivity loss.

**Effort to fix:** small for view-switching shortcuts; medium for full Linear-style coverage.

### 3.8. No dark mode

Linear, Vercel, GitHub, Stripe Dashboard all default-dark or offer it. Operators stare at this all day. Tailwind + the `--leasium-*` CSS variables make a dark theme cheap. The design source of truth doesn't currently spec dark colours — that's the only blocker.

**Effort to fix:** small (add dark token values to the design source of truth + a `.dark` class on the body) to medium (audit every component for dark contrast).

### 3.9. Notion-style multi-view absent

Properties table is fixed as a table. Operators looking at the SKJ portfolio might want:
- **Map view** — properties pinned on a Brisbane map.
- **Board view** — columns by status (Leased / Partial / Vacant).
- **Calendar view** — lease expiries plotted over the year.

This is differentiating but expensive. Skip for v1; flag as future.

### 3.10. Mobile and density

The app shell uses `min-[1600px]` media queries — it's optimised for wide monitors. The user picks Leasium up on a phone occasionally for site visits or contractor calls; the current responsive story is probably "shrinks but doesn't reflow well". Worth a real audit if mobile use becomes regular.

**Effort to fix:** medium-large depending on how many surfaces need rework.

## 4. Recommended next moves

Ranked by leverage for SKJ's internal-use period. Each lists effort, behaviour change, and risk.

### Tier 1 — ship in the next 2-3 weeks

**(a) Convert top-bar nav to a left sidebar.**
*Effort: medium (1-2 days).* *Risk: low (purely visual; routes unchanged).*

Build a `<AppSidebar>` component, move nav items in, kill the wrapping top row. Frees up the top bar for: workspace switcher, environment chip, Cmd+K, account menu, future "Ask" widget. Matches the design source of truth.

**(b) Build a reusable `<DetailDrawer>` and migrate Properties + Tenants onto it.**
*Effort: medium (2-3 days).* *Risk: low.*

Generic drawer with sticky header, tabbed content, primary-action footer. Property row click opens drawer instead of navigating. Tenant row same. Maintenance already has its pattern — converge on the generic version. Cuts navigation cost for every operator interaction.

**(c) Add Stripe-style trend deltas + sparklines to dashboard metric cards.**
*Effort: small (1 day).* *Risk: low.*

Each `DashboardMetricCard` already shows a count + chip + nextAction. Add an optional `delta: { direction, percent }` and `sparkline: number[]` to render a 7- or 30-day trend. Data already exists (rent roll history, maintenance counts, etc.); add an `/api/v1/dashboard/metric-trends` endpoint that returns the deltas.

**(d) URL-persistent filters across Properties, Tenants, Operations.**
*Effort: small (1 day).* *Risk: low.*

Sync filter pill state to query params (`?occupancy=vacant&owner_tag=skj`). Operators can bookmark or share filtered views. First step toward saved-views v2.

### Tier 2 — ship over the next 1-2 months

**(e) AI Q&A surface — "Ask Leasium".**
*Effort: larger (1-2 weeks, 3-4 commits).* *Risk: medium (LLM quality, citation accuracy).*

Floating widget (or inside Cmd+K). Backend `/api/v1/ai/ask` does retrieval over the operator's entity context (active leases, upcoming events, recent maintenance, recent invoices), passes to an LLM with strict citation requirement. Limits to read-only questions in v1 — no provider mutations.

Differentiator against Re-Leased Credia Advise on Australian-specific tenancy/Xero context.

**(f) Cross-property activity feed.**
*Effort: medium (2-3 days).* *Risk: low.*

Backend `/api/v1/activity/recent?entity_id=X&limit=30` aggregates from audit logs. Renders as a small "Recent activity" panel under the dashboard metric cards. Vercel-style "what changed since you looked."

**(g) Dark mode v1.**
*Effort: small-medium (1-2 days).* *Risk: low (visual regressions in unaudited components).*

Add dark token values to the design source of truth. Apply `.dark` class via system preference + a toggle in the account menu. Audit the 5 most-used surfaces (Dashboard, Properties, Tenants, Operations, Notifications) for contrast.

**(h) Keyboard shortcuts beyond Cmd+K.**
*Effort: small (1 day).* *Risk: low.*

Add `G D` / `G P` / `G T` / `G O` / `G B` / `G N` / `G S` for the 7 nav items. Add `C` for primary create action per surface. Add `?` to show the shortcut cheatsheet. Document in the design source of truth.

### Tier 3 — bigger bets, post-internal-period

**(i) Inline-editable tables.**
**(j) Saved views per operator.**
**(k) Notion-style multi-view for Properties (map / board / calendar).**
**(l) AI inbox processor (Credia Action equivalent).**
**(m) Mobile-first audit pass.**

All of these are valuable but heavier and arguably more relevant once SKJ has external customers or larger internal teams.

## 5. What to NOT do during the internal-only window

These are valid investments but explicitly deprioritised right now per [`leasium-internal-first-6-months`](../../../Library/Application%20Support/Claude/local-agent-mode-sessions/...) memory:

- **Marketing landing-page polish.** No external sales = no incremental signups to convert.
- **Branded communication template editor UI.** Foundation table shipped, but SKJ as an internal user doesn't need to edit defaults.
- **Tenant portal visual redesign.** Tenants aren't using it externally yet.
- **Customer-faced documentation / help centre.** Operators can ping each other; no support volume to deflect.

Defer all of these to month 5-6 when SKJ starts thinking about a wider user base.

---

## 6. The single biggest "do little, get a lot" move

If only one item from this whole review ships, it should be **(e) AI Q&A — "Ask Leasium"**.

Reasoning:
- It directly delivers the brand promise ("Lease operations, automated").
- It compresses 10-15 minutes of clicking into a 5-second question (matching the Credia/AppFolio time-saved claims).
- It uses infrastructure that's already in the codebase (OpenAI key, source citations, audit trails, the read-first culture).
- The competitive moat against Re-Leased Credia is *Australian-specific context* (Xero integration, AUD currency, Australian tenancy law cadence) — which Leasium is closer to than any global competitor.
- It scales naturally: today Q&A only; later, suggested actions; later, autonomous tasks (with operator approval).

Everything else in this review is incremental polish. This one is potentially the inflection point between "useful internal tool" and "platform you'd let SKJ Capital depend on for a $X00M portfolio."
