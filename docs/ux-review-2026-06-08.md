# Leasium UX review — live walkthrough of leasium.ai

Date: 2026-06-08
Method: signed-in operator walkthrough of the production app at 1512px (Dashboard, Properties, tenant record, Operations Queue + Compliance, Smart Intake, Money, Billing Readiness, Insights, Notifications, Settings), cross-checked against ui-ux-pro-max design-intelligence rules (data-dense dashboard pattern, WCAG/HIG/MD guidelines).
Status: review report only — no code changed. Each item below is a candidate ticket; design-facing fixes remain Remba-pending per docs/design-governance.md (currently prototype mode).

---

## P0 — Fix first (trust, safety, dead ends)

### 1. Property title click-to-edit has no affordance and no guard
On /properties, clicking a property *title* instantly turns it into an inline rename input (with ✓/✕). There is no hover cue, no pencil-on-title, and a stray keystroke renames a real portfolio record. This is the known "safer edit affordances" gap, and it is the most dangerous interaction found.
Recommendation: row click = select/open; rename only via the existing row pencil or detail view, with explicit save. (Rule: confirmation/intent before mutation; severity High.)

### 2. /smart-intake is a raw Next.js 404 with no way back
The nav route is /intake, but the obvious guessable URL /smart-intake returns the unbranded black Next 404 — no logo, no nav, no link home. Any typo'd or stale URL strands the operator.
Recommendation: branded 404 with "Back to Dashboard" + redirect /smart-intake → /intake. (Rules: escape-routes, error-recovery.)

### 3. Conflicting headline numbers across surfaces
The same portfolio shows: Dashboard "Operations 20 / Billing blockers 44", Operations KPI strip "Urgent 0 / Awaiting 0 / Arrears 0 / Disputed 0 / Reminders 0" above a queue of 65 open items, Insights "Live Exceptions 12", Billing Readiness "Readiness blockers 173 / Missing Xero mapping 142 / Xero issues 50". These are different measures, but the labels don't say so — it reads as the product disagreeing with itself.
Recommendation: one named metric vocabulary (e.g. "Open work", "Urgent work", "Billing blockers", "Readiness checks"), reused verbatim on every surface; if a card is a different cut, the label must carry the qualifier. The Operations strip of five zeros above 65 open items is the worst case — those cards should reflect the queue the page actually shows.

### 4. Billing Readiness repeats identical blocker rows instead of grouping
Under one tenancy, "Xero mapping needs review" renders 4+ times with identical title, identical description, identical "Map Xero" button. With 142 missing-Xero-mapping items the page becomes hundreds of duplicate cards. Insights has the same disease ("Auto & General Lvl 2-3 (combined)" three times; "Tenant onboarding ready for review" twice).
Recommendation: group by (tenancy × issue type) with a count — "4 Xero mappings need review → Map Xero" — and de-duplicate Insights exception rows. (Rule: bulk actions over repeated per-row actions.)

## P1 — High-value structure and density fixes

### 5. Stale dates presented without alarm
Billing action queue shows "active lease, next due 07 July 2023" and "next due 01 May 2025" in calm grey strips. A next-due three years in the past is itself a data-quality blocker and should look like one.
Recommendation: overdue/next-due-in-the-past gets a red "stale" badge and ideally its own blocker type. (Rule: color + icon + text for functional states.)

### 6. Badge stacking is redundant and sometimes contradictory
Work rows show "7d overdue" + "due soon" + "Critical date" together; compliance rows show "In 2d" + "Due soon". Overdue and due-soon are mutually exclusive; "Critical date" is the row's type, not a status.
Recommendation: one status chip per row (overdue XOR due-soon XOR upcoming) + one muted type label. Frees width and makes scanning honest.

### 7. Operations toolbar is three filter systems deep
Queue page stacks: section tabs (Queue/Maintenance/Compliance/Arrears) → CSV/digest button row (6 controls) → bucket chips (Open/Unassigned/Assigned/Follow-up/My work) → a separate "All open work" dropdown floating right. Two unrelated selects sit in opposite corners of the same card.
Recommendation: keep tabs + one chip row; collapse CSV/digest actions into an overflow "Export / Digest" menu; merge the dropdown into the chip row. (Rules: overflow-menu, avoid mixed filter patterns. Notifications has the same 4-export-button row — same fix.)

### 8. Waive/Complete and Deactivate carry equal visual weight
Every work row pairs "Complete" and "Waive" as twin outline buttons; Settings repeats a red "Deactivate" button on all 9 operator rows. Semi-destructive actions sit at the same prominence as the primary action, multiplied per row.
Recommendation: primary action visible, destructive/secondary actions behind a row "…" menu with confirm. (Rules: destructive-emphasis, destructive separation, confirmation-dialogs.)

### 9. Money hub is a menu page
/money renders one launcher card ("Open Billing Readiness") and a guardrail note — an extra click that conveys no state. The Statements/Xero/Basiq pills presumably hide similar launchers.
Recommendation: make the hub show live state per area (blockers count, last Xero sync, Basiq feed status) with the link as the card action — same KPI-card pattern already used elsewhere — or fold Money into Billing Readiness until there is real content.

### 10. Empty-state-dominated layouts on Compliance tab
/operations?tab=compliance gives roughly two-thirds of the viewport to two large empty panels (Recurring checks, Smart Intake reviews) while the populated Linked obligations list is squeezed into half a column.
Recommendation: collapse empty panels to a single-line row with the explainer ("No recurring checks yet — fire safety, insurance… will appear here") and let populated content take the width. (Rule: content-priority.)

### 11. Properties page bottom is a void; End-key lands on blank screen
The portfolio page scrolls past the last row into more than a full viewport of empty background (pressing End shows a fully blank screen). Likely a reserved-but-unrendered detail region. Also: clicking a row body gives no visible response — the selected-property detail (if any) is nowhere near the click.
Recommendation: remove the reserved space; if a row click selects a property, the response must be visible (side panel or in-row expansion), not below 20 rows of list. (Rules: content-jumping/reserve-space, immediate feedback within 100ms.)

## P2 — Polish and consistency

### 12. Truncation without ellipsis or recovery
Owner chips cut mid-word ("GRHQ Pty Ltd (trustee for GRHQ Unit Tr", "SMSF › SJI No 1 (sublet) › Gorilla Rush 2"); Recent activity titles cut mid-word ("…should be dismissed and not appli") and then repeat the same truncated string in the description line. People rows show "invoices@advancedsupportservices.…." with four dots.
Recommendation: CSS ellipsis + title tooltip everywhere; never render title and description as the same string; activity rows should show document type + short name, not raw extracted titles. (Rule: truncation-strategy.)

### 13. Stat-card pattern misapplied to words
Settings renders "clerk", "Enforced", "Ready" in the 32px big-number style. Reads as broken data, and lowercase "clerk" leaks an internal enum.
Recommendation: status words get a chip + label layout; reserve big-number cards for numbers. Same family: Notifications KPI cards repeat their own label as a chip inside the card ("UNREAD 1 [Unread]") — drop the chip.

### 14. Repeated-identity noise
People rows and Settings operator rows show the email twice (name slot falls back to email, then email again beneath; contact email and billing email duplicated when identical). Tenant record header buttons put "Delete tenant" directly beside "Edit profile".
Recommendation: show email once when it is the display name; "Billing: same as contact" when equal; move Delete tenant behind the record "…" menu. (Rule: destructive-nav-separation.)

### 15. Copy and casing nits
"1 notices" (pluralisation); lowercase sentence fragments joined with periods ("Entity is not connected to Xero. base rent is missing…"); TYPE column values lowercase ("commercial office", "other") while every other label is sentence-cased; raw template keys ("work_assignment_notification v1") and raw phone numbers surfaced in Notifications despite the named-template catalog existing — map keys to their plain titles there too.

### 16. Dashboard "Operations 20 · vs last week" card
The dangling "· vs last week" with an empty sparkline area looks unfinished; either render the trend or drop the comparator text. Dashboard "Upcoming lease events" rows titled "Tenant onboarding due / Sent" with an empty "-" badge need a real status and a tenant name ("Tenant onboarding - Tenant" twice tells the operator nothing).

### 17. Dashboard ↔ Smart Intake duplication
The Dashboard embeds the full Smart Intake drop zone + review queue, and Smart Intake embeds the Dashboard's four KPI cards. Each surface should keep its own job: Dashboard links to intake (count + CTA), Smart Intake owns upload/review. Reduces both pages' length and keeps "where do I do X" unambiguous.

## What's working well (keep)

- The two-spine IA (Properties / People) with the 7-item sidebar is clean and matches the DoorLoop-legibility goal; nav active states are clear.
- Tenant record page shape (header → tabs → stat strip → panels) is a strong, reusable template — extend it unchanged to Vendor/Owner/Property records.
- Review-first guardrail copy ("Provider actions remain review-first…") is consistent and confidence-building; keep it as quiet single-line strips, not stacked double explainers (the tenant Correspondence panel currently stacks two).
- Urgency bucketing on the Operations queue (Overdue / Due soon / Scheduled with Hide/Show) is the proven "long page" fix working as intended.
- Status colour language (red overdue / amber soon / green ok) is consistent across all surfaces.
- Empty states almost always say what will appear and why — well ahead of baseline.

## Suggested ticket order
